# Contributing to voxassist

Thanks for your interest in contributing to **voxassist**!

## How to contribute
We use the **fork + pull request (PR)** workflow:

1. Fork the repository.
2. Create a feature branch in your fork.
3. Make your changes.
4. Open a PR back to this repository.

## Development setup (Node.js)
1. Install Node.js (prefer the current LTS release).
2. Install dependencies:
   - `npm install`
3. Run the project locally (if applicable):
   - `npm run dev` (or the project’s documented start script)

## Required checks
Before your PR can be merged, it should pass the project’s automated checks (for example: linting, tests, and build). If checks fail, please update your PR until they pass.

## Commit messages
Write clear, descriptive commit messages. If your change is non-trivial, consider explaining the “why” in the PR description.

## Reporting bugs / requesting features
- Use GitHub Issues to report bugs or request enhancements.
- Include enough context for maintainers to reproduce or evaluate the request.

## Security vulnerabilities
Please **do not** open a public issue for security problems.

Instead, report vulnerabilities using **GitHub Private Vulnerability Reporting / Security Advisories** (a private advisory) so maintainers can coordinate a responsible disclosure.

## Code of Conduct
If this repository has a Code of Conduct (for example, `CODE_OF_CONDUCT.md`), contributions and community interactions are expected to follow it.

If one is not present yet, please still be respectful and constructive in all discussions.
